Dựa trên định hướng mới: [read file specs/IDEA.md] và Codebase hiện tại của tôi (đã đính kèm/index).

Nhiệm vụ của bạn:

Audit: Liệt kê các Module/File cũ sẽ bị ảnh hưởng trực tiếp bởi thay đổi này.

Impact Analysis: Những thay đổi này có gây 'breaking changes' cho API hoặc Database hiện tại không? Nếu có, hãy đề xuất phương án Migration.

Refactor Spec: Soạn thảo một tài liệu specs/REFACTOR.md chi tiết. Phân chia rõ:

Keep: Những gì giữ nguyên.

Modify: Những gì cần sửa (nêu rõ logic cũ vs logic mới).

Delete: Những gì cần loại bỏ hoàn toàn.

Technical Spec: Cập nhật lại tài liệu specs/PRD.md cho các Agent thực thi ở bước sau.
